# nanobot-eino 架构概览

## 项目简介

nanobot-eino 是一个基于 [Eino SDK](https://github.com/cloudwego/eino) 构建的 AI Agent 框架，用 Go 语言编写。它实现了与 Python nanobot 相同的能力：多模型支持、工具调用、记忆管理、会话持久化、多通道消息收发。

**模块路径**: `github.com/wall/nanobot-eino`

## 核心依赖

| 依赖 | 用途 |
|------|------|
| `github.com/cloudwego/eino` | Eino Agent/React 框架，提供 LLM 调用、工具调用、流式响应的核心能力 |
| `github.com/spf13/cobra` | CLI 命令行框架 |
| `github.com/spf13/viper` | 配置文件加载（YAML/JSON/TOML） |
| `github.com/larksuite/oapi-sdk-go/v3` | 飞书（Lark）开放平台 SDK |
| `github.com/robfig/cron/v3` | Cron 定时任务调度 |
| `github.com/modelcontextprotocol/go-sdk` | MCP（模型上下文协议）客户端 |

## 核心数据流（一次完整的对话处理）

```
1. 用户消息 → 通道适配器(Feishu/CLI)
                │
                ▼
2.          消息总线 (Inbound channel)
                │
                ▼
3.       RunInboundLoop (按 Session Key 分配 Worker)
                │
                ▼
4.         Agent.ChatStream(sessionID, input)
                │
        ┌───────┴────────┐
        ▼                ▼
5.  MemoryConsolidator   Prompt Loader
    (上下文裁剪)         (构建系统提示词)
        │                │
        └───────┬────────┘
                ▼
6.      buildMessages → [System + Skills + Memory + History + User]
                │
                ▼
7.      React Agent Loop (Eino)
        ┌─────────────────────────┐
        │ LLM 思考 → 工具调用      │
        │   → 工具执行 → 结果回传   │
        │   → LLM 继续思考 → ...   │
        │   → 最终文本输出         │
        └─────────────────────────┘
                │
        ┌───────┴────────┐
        ▼                ▼
8.  流式输出到用户    保存会话 + 记忆整合
```

## 关键设计决策

1. **消息总线解耦**: Agent 和通道之间通过 `MessageBus`（两个 Go channel）通信，Agent 不直接知道消息来源
2. **按 Session 串行化**: `RunInboundLoop` 为每个 session 启动独立 goroutine，同一 session 的消息排队处理，不同 session 并行
3. **MCP 延迟加载**: MCP 工具在首条消息时才连接，避免启动时因 MCP 服务不可用而失败
4. **记忆分层**: 短期记忆=会话历史，长期记忆=MEMORY.md（覆盖写），历史记录=HISTORY.md（追加写）
5. **工具包装器**: 所有工具通过 `WrapTools` 统一包装，提供结果截断（16KB）和进度回调
